Project 1: Chess but we have PE

Assignment 1: Not-a-button

Ideation

For this assignment, I had an initial idea to use my tattoo as a button. My computer would recognize my tattoo with the web camera and would then open a webpage. After approximately 5 hours of trying to make a reliable system, I realized that it would be to hard for me to do, so I scrapped my idea.

My second idea I got was to create a chess game where you would select the square by moving your body along a ultrasonic distance sensor. Using two ultrasonic distance sensors would make two players collaborate with each other to choose a square and move the piece. The initial sketch can be seen on the right.


My initial sketch

Prototyping - Digital

To start off, I used digital tools to try the concept out. I did this for two reasons. First of, it is easier for me to create digital artifacts due to my disability. Second of all, I did not have access to an Arduino since I did not own any and every Arduino in MIDDLA was taken already.

I started using TinkerCAD, an online CAD service, since they have an Arduino simulator and a bunch of sensors and actuators that can be simulated. It also allowed Arduino code, which enabled me to produce working code that would be functional on an actual Arduino device. I created a simple circuit that connected two Ultrasonic Distance Sensors and wrote some code that would measure the distance in centimeters and map these to return a value between 1 and 8 (since there is 8 rows and 8 columns on a chess board). You can play with the circuit and the code here. (An account on TinkerCAD is needed)

In addition to the circuit, I wrote a really simple Processing script that simply drew a chess board and highlighted the selected square. Since I could not connect the Arduino simulator to Processing, I could not do anything more with it than just showing the concept.


The first version of the Processing sketch

Prototyping - Gone physical

While staying digital would be sufficient for the course, I felt like the interaction could not be experienced and therefore the hand in felt boring. Therefore I bought an Arduino to be able to implement it physically. I brought the Arduino to MIDDLA and Andreas helped me connect the circuit I made in TinkerCAD. However, there was only one Ultrasonic sensor available, so I had to work with that. I edited the code so I could send the data to Processing. You can see the interaction on the next slide.

I did some final polishing on the Processing sketch to make sure it looked nice. I also made it recieve data from the Arduino to be able to use that in my Processing sketch. Since I only had one sensor, I made that so you could move vertically and horisontally by implementing a simple timer that would change how the input would be used (vertically or horisontally). It was definately more janky than two sensors but it had to suffice due to the lack of sensors


The final version of the Processing sketch

Result and design critique

For the result, I am quite happy with the result. There is some issues with the sensor since it did not read always read the values accurately. There are mainly two potentional issues that could cause this. I did not do any data processing such as sampling, which could cause to inaccuracies. The sensor could also be broken, but the one I used was all new, so that should not be an issue unless it is a manufacturing issue. For a final version, this needs to be addressed.

The critique I got was good. The interaction felt interesting and self explainatory, which was nice to hear. I explained the original idea with two sensors, which they all understood. I got some critique about not using a proper ideation process, which would have helped me with my time. As mentioned, I tried to make an interaction with my tattoo, but when I realized that it was too inaccurate, I went back to square one and had no idea what to do. This chess idea was just an Heureika moment and I was just lucky that I got the idea. Otherwise, I'd probably be stuck and just trying to get something to hand in, which is a recipe for bad design. If I used proper ideation methods, I would have came up with several ideas that I could fall back to. I did not have this and was "just lucky" to get another doable idea in the time frame I had.
